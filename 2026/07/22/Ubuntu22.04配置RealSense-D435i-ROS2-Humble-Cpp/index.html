Ubuntu 22.04 配置 Intel RealSense D435i：ROS 2 Humble 与 C++ 实战

发表于2026-07-22|更新于2026-07-22|机器人视觉
|阅读量:

图 1：D435i 连接 Ubuntu 主机后，可同时提供彩色、深度、点云与 IMU 数据。

Intel RealSense D435i 是一款带 IMU 的双目深度相机，适合机器人导航、三维重建、目标定位和视觉惯性融合。本篇记录一套可以复用的配置流程：在 Ubuntu 22.04 上安装 RealSense SDK 与 ROS 2 驱动，启动深度、彩色和 IMU 数据流，最后用 ROS 2 C++ 编写订阅节点。

本文使用 Ubuntu 22.04（Jammy）与 ROS 2 Humble。Humble 对 Ubuntu 22.04 提供 Tier 1 支持，适合需要长期稳定环境的项目。

1. 环境与硬件准备

建议准备：

  • Ubuntu 22.04 64 位
  • ROS 2 Humble Desktop
  • Intel RealSense D435i
  • USB 3.0 数据线与 USB 3.0 接口
  • 至少 4 GB 可用磁盘空间

先更新系统。ROS 2 官方特别提醒，在 Ubuntu 22.04 安装 Humble 前应先更新系统，避免旧版 systemd 或 udev 引发依赖问题。

2. 安装 RealSense SDK 与 ROS 2 驱动

RealSense 官方建议只选择一种 SDK 安装方式，避免系统包与源码版本相互冲突。对于普通 Ubuntu 22.04 + ROS 2 Humble 项目，直接安装 ROS 软件源中的二进制包最省事。

什么时候需要从源码编译？

以下情况更适合从源码编译：

  • 需要官方仓库中的最新修复；
  • 二进制包不支持当前内核或设备固件；
  • 需要修改 RealSense ROS Wrapper；
  • 项目需要固定特定版本。

不要同时保留多个不同来源的 librealsense2，否则容易出现动态库版本不一致、节点启动后崩溃等问题。

3. 检查 USB 连接

插入 D435i 后检查设备：

lsusb -t 中相机最好工作在 5000M 的 USB 3.x 链路。如果显示 480M，通常表示相机连接到了 USB 2.0 接口、扩展坞或质量较差的数据线。USB 2.0 下高分辨率深度与彩色数据很容易丢帧。

9. 常见故障排查

9.1 相机没有被识别

依次检查 USB 接口、数据线和供电，尽量不要先经过扩展坞。

9.2 failed to set power state

通常与 USB 链路、权限或设备被其他程序占用有关。关闭 realsense-viewer 和其他相机节点，重新插拔设备后再启动。

9.3 帧率不足或持续丢帧

确认链路为 USB 3.x，并降低分辨率或帧率：

9.4 节点能启动但订阅不到数据

重点检查：

  1. 自定义节点是否加载了正确的 ROS 2 环境；
  2. 话题名称是否与 ros2 topic list 一致；
  3. 发布端与订阅端 QoS 是否兼容；
  4. 是否同时启动了多个 RealSense 进程。

9.5 动态库版本冲突

如果系统中同时存在源码安装和 Debian 安装的 SDK，应保留一种安装方式并清理另一种。
